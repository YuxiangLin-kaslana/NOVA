# 路线B headline 跑通:LLM 自举开放词表异常分类闭环

> 日期:2026-06-17 · A100(job 19320479)· gpt-4o-mini · `scripts/exp_openvocab_loop.py`

## 设定
受控合成(可控注入器 + 相关基底)。已知 5 类进检测器预训练;correlation_break **留出**,后段涌现。
agent 流式逐窗:检测器先判 → 疑似新类/不确定/审计才叫 LLM → LLM 命名新类 → 加类别(grow head)
→ 干净伪标签(类平衡)在线重训 → 检测器学会 → 不再叫 LLM。

## 结果(新类涌现后第二段)
| 指标 | frozen | bootstrap |
|---|---|---|
| 整体分类准确率 | 45.2% | **77.6%** |
| 新类 correlation_break | 0% | **66.3%** |
| 词表 | 5 | **6** |

新类准确率随时间: [0.40, 0.36, 0.75, 0.93, 0.89](爬升)
LLM 调用率随时间: [0.39, 0.57, 0.25, 0.11, 0.14](先升后降→审计底线)

## 结论
**无人工标注下系统自己发现+长出新异常类别,整体准确率 45%→78%,LLM 依赖随之衰减。** route B 核心 claim 成立。

## 关键工程坑(均为真实问题,已解决)
1. 闭集网对 OOD 新类**自信误判**(非"不确定")→ 发现不能靠检测器置信度 → 改用**证据判据**(无任何已知签名升高=疑似新类)。
2. 学新类导致**灾难性遗忘**已知类(整体准确率打平)→ **类平衡在线重训** + replay。
3. LLM 调用**截断静默失败**(max_output_tokens 太小,JSON 被截)→ 强制 JSON-only + 提到 400 tokens。
4. LLM **过度列举**多个概念 → got[0] 取到已知噪声 → 用证据判据滤出**非已知**概念当标签。
