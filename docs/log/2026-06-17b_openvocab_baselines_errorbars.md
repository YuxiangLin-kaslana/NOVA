# 路线B headline 加固:对照臂 + 多 seed 误差棒(Task A)

> 日期:2026-06-17 · A100(job 19342784)· gpt-4o-mini · `scripts/exp_openvocab_loop.py`(重构为 4 臂多 seed)

## 目的
把 2026-06-17 跑通的"LLM 自举开放词表闭环"做成可发表强度:加 **人工标注上界** 与 **LLM-only**
两个对照臂,并跑 **5 seeds** 给误差棒。受控合成、留出新类 `correlation_break` 后段涌现。

## 四个臂
- **frozen** 闭集检测器,永不扩词表,无 LLM(下界)。
- **bootstrap(本方法)** 证据闸门触发 → LLM **新类命名器**(列举概念取非已知)→ 扩词表 → 类平衡在线重训。
- **oracle(人工上界)** 同一闭环,闸门触发时用**真值**当标签(无 LLM)→ 衡量 LLM 标注噪声的代价。
- **llm_only(成本对照)** 后段每窗直接叫 LLM 做 **top-1 单标签分类**,无检测器学习 → 恒 100% 成本。

## 结果(后段=新类涌现后,mean±std over 5 seeds)
| 臂 | 整体准确率 | 新类 correlation_break | 标注成本 |
|---|---|---|---|
| frozen | 48.9% ± 2.4% | 0.0% | 0 |
| **bootstrap** | **74.7% ± 7.7%** | **57.2% ± 16.8%** | LLM 31.4%(随时间 43%→17%) |
| oracle | 97.0% ± 1.7% | 95.3% ± 3.7% | 人工 24.5% |
| llm_only | 21.2% ± 2.5% | — | 100% |

新类准确率随时间(5 段):`0.32 → 0.43 → 0.62 → 0.68 → 0.82`(爬升)
LLM 调用率随时间(5 段):`0.43 → 0.41 → 0.31 → 0.26 → 0.17`(学会即衰减)

## 论证(四点)
1. 闭集检测器对新类无能为力:整体 49%、新类 **0%**。
2. LLM 逐窗直接分类**很弱**:即便用公平的 top-1 prompt 也只有 **21%**(低于 frozen)——通用统计证据下,gpt-4o-mini 不是好的逐窗分类器。
3. **本方法把两者结合**:证据闸门决定何时叫 LLM、LLM 只做**新类命名**、再学一个参数检测器 → **75%**,**逼近人工上界 97%**,且 LLM 成本随学会衰减到 17%。
4. 与上界 97% 的差距(~22pt)**纯粹来自 LLM 标注噪声**——oracle 用真值标签即达 97%。方法上限高。

## 关键发现:top-1 prompt 对"新类命名"反而更差
中途试过让 bootstrap 也用 top-1 prompt(选"最显著"概念),结果 **74.7%→61.5%、新类 57%→29%**。
原因:`correlation_break` 是相关性**下降**,信号微弱,LLM 不会把它排为"偏离最强",于是漏掉新类。
"列举全部 + 取非已知"恰能捞出这个**唯一**非已知候选。→ **这正是多新类(B)的核心难点**:命名器依赖
"非已知候选唯一",多个新类时失效。**top-1 函数(`gpt_recognize_top1`)已落地,留作 B 的起点**,但
B 还需解决证据可分性(见 multi-type blocker)。故两个臂用**不同 prompt**:bootstrap=命名器,llm_only=top-1 分类。

## 方差 caveat
bootstrap ±7.7%(新类 ±16.8%):seed 3 为弱离群(新类仅 31.5%),其余 4 seed 新类 44–75%。闭环偶尔
误命名导致单 seed 崩。oracle 方差极小(±1.7%)印证:不稳定来自 LLM 标注,而非方法结构。

## 结论
Task A 完成。headline "无人工标注下自举长出新类、准确率回升、LLM 成本衰减"在 5 seed 下稳定成立,
并有**人工上界(97%,高天花板)**与 **LLM-only(21%,逐窗 LLM 不够)**两个对照夹住。下一步:路线 B
(干净可分的多新类基准)。
