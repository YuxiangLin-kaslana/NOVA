# 路线B 的桥跑通:开放词表闭环让检测器"检测到"从未见过的新异常类型

> 日期:2026-06-17 · A100(job 19345375)· gpt-4o-mini · `scripts/exp_detection_tie.py`(5 seeds)

## 动机
A/B 证明的是开放词表**分类**(给异常命名)。审稿人核心问题:这对**检测**(二分类 normal vs anomaly)
有用吗?闭集检测器遇到从未见过的异常类型,到底是漏检、还是仍能报警只是类型错?本实验直接实测。

## 设定
判别式检测器含一个 **normal 类** + 3 个已知异常类(spike/level_shift/oscillation)。检测 = argmax≠normal。
流:50% normal + 50% 异常;后段新异常类型 **correlation_break** 涌现。证据门控(z>NOVEL_Z 且主导非已知签名,
normal 的 z 小自然被排除)发现新类 → LLM 命名 → 扩类 → 类平衡在线学。复用 `sigla_exp.ovbench` 正交基准。

## 结果(涌现后,mean±std over 5 seeds)
| 指标 | frozen | bootstrap |
|---|---|---|
| **新类检测召回**(argmax≠normal) | **15% ± 30%**(5 seed 中 4 个 =0%) | **88% ± 6%** |
| 整体检测 F1(anomaly vs normal) | 0.82 ± 0.06 | 0.87 ± 0.03 |
| 整体检测召回 | 0.78 ± 0.08 | **0.97 ± 0.02** |
| 整体检测精度 | 0.91 ± 0.17 | 0.80 ± 0.05 |
| 新类**分类**准确率 | 0% | 59% ± 5% |

LLM 调用率 33%±5%,词表长全率 100%,新类检测召回随时间 `[0.83, 0.86, 0.93, 0.89]`。

## 关键发现
**闭集判别检测器把 correlation_break 自信地判成 normal → 完全漏检**(5 seed 里 4 个新类检测召回 = 0%,
均值 15%)。这比"误分类成已知类"更严重:它根本**检测不到**这个从未见过的异常。
**开放词表闭环把新类检测召回从 ~0% 恢复到 88%,无人工标注**,且新类既被检测(88%)也被部分正确命名(59%)。

→ 直接坐实论文桥接主张:**闭集检测器对新型异常是盲的;只有 LLM 能识别并自举它们**,从而对从未见过的
异常类型实现检测/预警。整体召回 0.78→0.97 的提升几乎全部来自把"漏检的新类"救回来。

## 诚实 caveat
- 整体**精度** 0.91→0.80:在线重训 + 新类引入少量 normal 误报。整体 F1 仍升(0.82→0.87),但精度是代价。
- 为何闭集会判 normal(而非误报已知异常):correlation_break 是相关性破坏,既不像 normal 也不触发任何已知
  异常签名 → 判别式网络在 OOD 上落到"最近"的 normal。这与 route A 的"闭集网对 OOD 过度自信"一致。
- seed 3 例外(frozen 新类召回 74%):该 seed 预训练后 normal 边界恰好把部分 corr_break 划到异常侧——故方差大(±30%)。
